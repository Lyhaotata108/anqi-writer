# 内容导入接口文档

## 概述
本文档提供三个用于内容导入的 API 接口，包括获取明星关键词、获取分类列表以及发布文章。所有接口均需使用统一令牌（`token`）进行鉴权。

---

## 1. 获取明星关键词列表

### 接口描述
获取所有可用的明星关键词数据，用于后续文章发布时关联 `keyword_id`。

### 请求信息
- **URL**：`GET https://manage.teiastyle.com/import/keywords`
- **鉴权方式**：URL 参数 `token`

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| token  | string | 是 | 导入专用令牌，例如 `anqicms-import` |

### 响应格式
成功时返回 HTTP 状态码 `200`，响应体为 JSON 格式：

```json
{
    "code": 200,
    "data": [
        {
            "id": 199,
            "title": "fungus",
            "category_id": 17,
            "language": "en"
        },
        {
            "id": 198,
            "title": "MF Blocking Sticker",
            "category_id": 20,
            "language": "en"
        }
        // ... 更多关键词
    ]
}
```

### 响应字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| code   | int  | 状态码，200 表示成功 |
| data   | array | 关键词对象列表 |
| data[].id | int | 关键词唯一 ID（用于发布文章时传入 `keyword_id`） |
| data[].title | string | 关键词名称 |
| data[].category_id | int | 该关键词所属的分类 ID |
| data[].language | string | 语言代码，如 `en`（英文） |

---

## 2. 获取分类列表

### 接口描述
获取所有内容分类信息，用于文章发布时指定 `category_id`（当未提供 `keyword_id` 时使用）。

### 请求信息
- **URL**：`GET https://manage.teiastyle.com/import/categories`
- **鉴权方式**：URL 参数 `token`

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| token  | string | 是 | 导入专用令牌，例如 `anqicms-import` |

### 响应格式
成功时返回 HTTP 状态码 `200`，响应体为 JSON 格式：

```json
{
    "code": 200,
    "data": [
        {
            "id": 1,
            "type": "article",
            "title": "减肥药",
            "template_dir": "keto"
        },
        {
            "id": 5,
            "type": "article",
            "title": "CBD",
            "template_dir": "cbd"
        }
        // ... 更多分类
    ]
}
```

### 响应字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| code   | int  | 状态码，200 表示成功 |
| data   | array | 分类对象列表 |
| data[].id | int | 分类唯一 ID（用于发布文章时传入 `category_id`） |
| data[].type | string | 内容类型，如 `article` |
| data[].title | string | 分类名称 |
| data[].template_dir | string | 模板目录标识，用于前端渲染 |

---

## 3. 发布文章

### 接口描述
创建一篇新文章，支持 Markdown 格式的内容，并自动关联关键词或分类。

### 请求信息
- **URL**：`POST https://manage.teiastyle.com/import/article`
- **鉴权方式**：URL 参数 `token`
- **请求头**：`Content-Type: application/json`
- **请求体**：JSON 格式

### 请求参数（Body JSON）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| title | string | **是** | 文章标题 |
| content | string | **是** | 文章正文，使用 **Markdown 格式**（详见下方内容说明） |
| keyword_id | int | 条件必填 | 明星关键词 ID（来自接口 1）。若提供此字段，可自动获取对应的 `category_id`，此时 `category_id` 可省略 |
| category_id | int | 条件必填 | 分类 ID（来自接口 2）。当未提供 `keyword_id` 时，必须手动传入 `category_id` |
| keywords | string | 否 | 自定义关键词标签，多个用英文逗号分隔，例如 `"关键词1,关键词2"` |
| description | string | 否 | 文章摘要描述，若不填系统可能自动截取 |

> **注意**：`keyword_id` 和 `category_id` 必须至少提供其中一个。优先推荐使用 `keyword_id`，因为系统会自动补全分类。

### 内容字段（`content`）特殊说明
- 内容为 **Markdown 格式**，支持标准 Markdown 语法。
- **图片**：使用占位符图片，格式为 `![Image description](image-placeholder.png)`，系统会后续替换为实际图片地址。
- **视频**：目前仅支持 YouTube 嵌入，使用 `iframe` 占位符，示例如下：
  ```html
  <iframe width="795" height="448" frameborder="0" allowfullscreen src="youtube-url-placeholder"></iframe>
  ```
  其中 `src` 属性填写占位符 `youtube-url-placeholder`，系统将在处理时替换为实际 YouTube 链接。

### 请求示例
```json
{
  "title": "The Benefits of Fungus in Skincare",
  "content": "## Introduction\n\nFungus extracts have shown promising results...\n\n![Fungus extract](image-placeholder.png)\n\nWatch the full explanation:\n<iframe width=\"795\" height=\"448\" frameborder=\"0\" allowfullscreen src=\"youtube-url-placeholder\"></iframe>",
  "keyword_id": 199,
  "keywords": "fungus,skincare,natural",
  "description": "Discover how fungus can improve your skin health."
}
```

### 响应格式
成功时返回：

```json
{
    "code": 200,
    "msg": "保存成功",
    "data": {
        "id": 1
    }
}
```

### 响应字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| code   | int  | 状态码，200 表示成功 |
| msg    | string | 操作结果描述 |
| data.id | int | 新创建的文章 ID |

---

## 错误处理
若请求失败（如参数缺失、token 无效、关联 ID 不存在等），接口将返回非 `200` 的 `code`，并附带 `msg` 说明错误原因。请根据错误信息调整请求。

## 使用建议
1. 先调用接口 1 或 2 获取有效的 `keyword_id` 或 `category_id`。
2. 推荐优先使用 `keyword_id`，简化分类选择逻辑。
3. 发布文章前，确保 `content` 中的占位符格式准确，以便系统正确替换。
